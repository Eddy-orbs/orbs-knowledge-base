# Staking Guide

## Summary
이 문서는 Orbs PoS 참여를 위한 실전 안내 문서입니다. 기술 개념 설명보다, 실제로 어떻게 참여하고 무엇을 확인해야 하는지에 초점을 둡니다.

## Who Can Participate?
### Delegator
- ORBS를 stake하고 Guardian에게 위임하는 일반 참여자
- rewards, split, lockup, Guardian 신뢰도와 운영 상태를 이해해야 함

### Guardian
- 네트워크 운영/보안에 직접 참여하는 validator 역할
- 더 높은 기술적/운영적 책임을 가짐
- node 운영과 troubleshooting 문서 이해가 필요함

## Delegator Flow
1. ORBS 보유 및 지원 네트워크/지갑 준비
2. TETRA 또는 관련 공식 도구 접근
3. Guardian 후보 확인
4. stake 및 delegation 진행
5. rewards / lockup / status 확인
6. 필요 시 undelegation / unlock 절차 확인

## Delegator Checklist
- Guardian 평판과 운영 안정성을 확인했는가?
- rewards split 구조를 이해했는가?
- lockup/unlock 기간을 확인했는가?
- Ethereum/Polygon 등 관련 네트워크 수수료를 고려했는가?
- analytics / status 페이지에서 운영 상태를 확인했는가?

## Guardian Flow
1. 공식 Guardian 가이드 검토
2. 운영 요건 및 책임 이해
3. node / monitoring / troubleshooting 문서 숙지
4. Guardian 참여 절차 진행
5. 지속적 운영 상태 모니터링

## Important Cautions
- reward, APR, split, lockup 정보는 PoS 버전에 따라 다를 수 있음
- official FAQ snippet references a 14-day locking period in the PoS V3 context, but version scope를 명확히 확인하고 설명해야 함
- 2020년 전후 locking mechanism 변화가 있었으므로 historical staking guidance와 current staking guidance를 혼동하지 말아야 함
- stake 전에 Guardian 신뢰도와 운영 상태를 확인하는 것이 중요
- 실시간 데이터는 analytics/status source를 우선 참고
- market site 수치보다 official analytics / status / docs를 우선시할 것
