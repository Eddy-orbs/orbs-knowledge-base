# Technical Questions

### Q: What is Orbs' current technical identity?
A: Under current official positioning, Orbs is closest to a decentralized Layer-3 or execution layer for DeFi protocols.

### Q: Why is Orbs described as a supplementary execution layer?
A: Public materials describe Orbs as complementing native smart contracts by handling more complex logic and execution through an external decentralized backend model.

### Q: What is the difference between Orbs VM and Orbs Lambda?
A: Based on public materials, Orbs VM is a heavier and more flexible Docker-container-based execution environment, while Orbs Lambda is a simpler serverless-style execution service.

### Q: What are virtual chains?
A: In GitHub and early technical materials, virtual chains appear as a core architectural concept in Orbs. This topic still needs deeper technical documentation.

### Q: What is Helix consensus?
A: It appears in GitHub and specification materials alongside virtual chains and is an important keyword for understanding the early network architecture. A fuller explanation still requires more GitHub and spec-driven work.

### Q: How can Guardians receive real-time Orbs node status alerts?
A: User-provided guidance: use the Telegram node-status alert bot @orbs_node_monitor_bot. It can monitor a specific node address or all nodes and notify when a node goes down.
