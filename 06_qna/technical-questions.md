# Technical Questions

### Q: Orbs의 현재 기술 정체성은 무엇인가?
A: 현재 공식 포지셔닝상 Orbs는 DeFi 프로토콜용 decentralized Layer-3 / execution layer에 가깝습니다.

### Q: Orbs는 왜 supplementary execution layer로 설명되는가?
A: 공개 자료상 Orbs는 native smart contract가 구현하기 어려운 복잡한 logic과 execution을 외부 decentralized backend 형태로 보완하는 역할을 합니다.

### Q: Orbs VM과 Lambda의 차이는?
A: 공개 자료 기준 Orbs VM은 Docker container 기반의 더 무겁고 유연한 실행 환경이고, Orbs Lambda는 더 단순한 serverless 스타일의 execution service입니다.

### Q: virtual chains는 무엇인가?
A: GitHub 및 초기 기술 자료 문맥상 Orbs는 virtual chains를 핵심 아키텍처 요소로 사용합니다. 이 부분은 기술 문서를 더 읽고 구체화할 필요가 있습니다.

### Q: Helix consensus는 무엇인가?
A: GitHub 스펙 자료에서 Orbs virtual chains와 함께 언급되며, 초기 네트워크 아키텍처를 설명할 때 중요한 키워드입니다. 자세한 기술 설명은 GitHub/spec 기반 보강이 필요합니다.
