# Response Policy

## Purpose
이 문서는 Orbs knowledge base 기반 응답 시, 정보가 없거나 시시각각 변하는 항목을 어떻게 처리할지 정리한 운영 가이드입니다.

## Core Rules
- knowledge base에 없는 세부 정보는 임의 외부 검색으로 메우지 않음
- 공식 자료에 없는 내용은 추측하지 않음
- 확인이 필요한 내용은 공식 contact/community 채널로 안내

## Exchange / Market Questions
- 거래소 상장 현황은 정적 리스트로 유지하지 않음
- 기본 안내 경로:
  - CoinMarketCap ORBS Markets
  - CoinGecko ORBS page
- 이유: 시장/상장 정보는 변동성이 높아 static docs에 고정하기 부적절함

## Missing Information Fallback
문서에 없는 정보가 들어오면 아래 순서로 처리:
1. knowledge base 내 문서 존재 여부 확인
2. 없으면 공식 contact/community로 안내
3. 필요 시 이메일 `hello@orbs.com` 안내
4. 국가별 커뮤니티가 필요하면 공식 커뮤니티 진입점으로 연결

## Contact Guidance
- Email: `hello@orbs.com`
- Community: official Orbs community entry points and country-specific communities when relevant

## Notes
- 이 정책은 FAQ, investor Q&A, 브리핑, 일반 질의응답에 공통 적용 가능
- live data와 static knowledge를 섞지 않는 것이 핵심
