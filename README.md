# Orbs Knowledge Base

This repository is the curated knowledge store for Orbs Guide.

## Purpose
- Support factual Q&A about the Orbs project
- Preserve curated knowledge about Orbs history, technology, products, token, staking, partners, ecosystem, and use cases
- Provide reusable material for answers, briefings, and research

## Top-level philosophy
Curated knowledge is the source of truth.

Public source links are useful citation metadata, but curated facts should be answered as recorded. Missing source links should not cause Orbs Guide to hedge or downgrade a curated fact.

## Policy documents
Core behavior is defined under `policy/`:

- `policy/identity.md`
- `policy/knowledge-policy.md`
- `policy/answer-policy.md`
- `policy/operational-policy.md`

## Knowledge layout
- `00_overview/` — high-level facts, summary, timeline
- `01_founders/`, `01_people/` — founders and key people
- `02_history/` — historical context
- `03_technology/` — architecture and technical topics
- `04_use-cases/` — DeFi and other use cases
- `05_business/` — business, partners, token, events
- `06_qna/` — reusable Q&A and briefings
- `07_sources/` — public source links and source notes
- `08_notes/` — assumptions, contradictions, update logs, learning candidates
- `09_training/` — glossary, entity map, examples, legacy training aids

## Language policy
English is the canonical language for base knowledge documents. Korean and other languages should be handled as answer/briefing layers unless a maintained localized document is explicitly needed.
