# Consensus / PoS

## Summary
Orbs publicly describes its consensus model as Proof-of-Stake (PoS). Public-facing materials also suggest that Orbs evolved its staking/security design over time, including earlier Ethereum-linked staking architecture and later Polygon-related staking expansion.

## How It Works (Current High-Level View)
- Orbs uses PoS for consensus/security.
- ORBS is publicly described as an ERC-20 token on Ethereum.
- Public Orbs materials indicate staking contracts are or were deployed on Ethereum.
- Public materials also reference multi-chain staking involving Polygon.

## Historical Evolution (Working Notes)
- PoS V1 appears tied to the mainnet-era design.
- PoS V2 is described publicly as "The Age of Guardians".
- Later materials reference multi-chain staking on Ethereum and Polygon.
- Additional exact milestone wording should be extracted from the V1/V2/V3 white papers before treating this as final.

## Validators / Guardians / Governance
- Public materials explicitly reference Guardians.
- Staking appears linked to electing Guardians and rewarding PoS participants.
- Exact validator/guardian relationship still needs whitepaper-based clarification.

## Security Considerations
- Public descriptions emphasize added security by using Ethereum as an external base/settlement/security layer.
- Need whitepaper confirmation on how Orbs explains the security guarantees in each version.
