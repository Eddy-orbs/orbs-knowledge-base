# Technical Comparisons

## Compare Against
- Ethereum
- EVM L1/L2 ecosystems
- Other PoS chains
- Pure smart-contract-only execution models

## Similarities
- Uses token-based incentive structure
- Uses Proof-of-Stake aligned validator / guardian model
- Interacts with EVM ecosystem and related chains

## Differences
- Orbs is not best described as "just another L1" in its current form
- Current positioning emphasizes supplementary decentralized execution rather than full-chain replacement
- Public materials stress that liquidity does not need to move to a new chain for Orbs products to be useful
- VM/Lambda style execution services and advanced order protocols give Orbs a more middleware / backend profile than many chain-native competitors

## Comparison Framing
The most useful way to compare Orbs is often not chain-vs-chain, but:
- native smart contracts only vs smart contracts + external decentralized execution layer
- DEX-only execution vs DEX + Orbs advanced execution services
- base-layer blockchain vs supplementary DeFi execution layer
