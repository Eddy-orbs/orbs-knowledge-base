# Use Cases Overview

## Summary
Orbs’ current use-case profile is clearly centered on DeFi execution infrastructure and AI / Agentic DeFi automation. Based on public materials, the main pillars are advanced trading orders, aggregated liquidity, decentralized derivatives, autonomous DeFi agents, and execution services.

## Main Categories
- Advanced trading orders
  - dTWAP
  - dLIMIT
  - dSLTP
- Liquidity optimization
  - Liquidity Hub
- Decentralized derivatives and perpetuals
  - Perpetual Hub
- Agent execution and AI-oriented DeFi automation
  - Orbs Agentic
  - The Orbs Agentic Challenge as a May 2026 adoption / trading campaign for Orbs Agentic usage
- General-purpose decentralized execution services
  - Orbs VM
  - Orbs Lambda

## Working Interpretation
Current Orbs appears much more focused on giving DeFi protocols and AI/agentic DeFi interfaces an external decentralized execution layer for complex order logic and execution logic that would be difficult to build directly into native smart contracts. In that sense, Orbs functions less like a user-facing application and more like a backend or middleware layer for DEXs, DeFi protocols, and autonomous DeFi agent workflows.

## Concise Korean Default Answer
- 오브스의 주요 유즈케이스는 DeFi 실행 인프라와 AI/Agentic DeFi 자동화입니다.
- 대표적으로 고급 주문 실행(dTWAP, dLIMIT, dSLTP), 유동성 최적화(Liquidity Hub), 퍼페추얼/파생상품 실행(Perpetual Hub)이 있습니다.
- 또한 Orbs Agentic을 통해 자율형 DeFi 에이전트가 의도를 실제 주문·스왑 실행으로 연결할 수 있고, Orbs VM/Lambda 같은 일반 실행 서비스에도 쓰입니다.

## Key Themes
- CeFi-level execution to DeFi
- Decentralized algorithmic orders
- Better liquidity sourcing and aggregation
- Intent-based derivatives and perpetuals
- Execution infrastructure for autonomous agents
- User-facing agentic DeFi adoption campaigns and trading competitions
