# Use Cases Overview

## Summary
Orbs’ current use-case profile is clearly centered on DeFi execution infrastructure. Based on public materials, the main pillars are advanced trading orders, aggregated liquidity, decentralized derivatives, and autonomous DeFi agents or execution services.

## Main Categories
- Advanced trading orders
  - dTWAP
  - dLIMIT
  - dSLTP
- Liquidity optimization
  - Liquidity Hub
- Decentralized derivatives and perpetuals
  - Perpetual Hub
- Agent execution and AI-oriented DeFi automation
  - Orbs Agentic
- General-purpose decentralized execution services
  - Orbs VM
  - Orbs Lambda

## Working Interpretation
Current Orbs appears much more focused on giving DeFi protocols an external decentralized execution layer for complex order logic and execution logic that would be difficult to build directly into native smart contracts. In that sense, Orbs functions less like a user-facing application and more like a backend or middleware layer for DEXs and DeFi protocols.

## Key Themes
- CeFi-level execution to DeFi
- Decentralized algorithmic orders
- Better liquidity sourcing and aggregation
- Intent-based derivatives and perpetuals
- Execution infrastructure for autonomous agents
