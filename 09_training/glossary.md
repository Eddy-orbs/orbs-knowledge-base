# Glossary

- Orbs: a decentralized Layer-3 project currently positioned around advanced trading and execution infrastructure for DeFi.
- ORBS token: the core token of the Orbs protocol, tied to fee settlement, staking, Guardian selection, and rewards.
- PoS: Proof-of-Stake, the core consensus and incentive structure of Orbs.
- Delegator: a participant who stakes ORBS and delegates voting weight to Guardians.
- Guardian: the validator-role participant set in the Orbs network.
- dTWAP: a decentralized time-weighted average price order method that executes large orders over time.
- dLIMIT: a decentralized limit-order method described as a special case of dTWAP.
- dSLTP: a decentralized stop-loss and take-profit order protocol.
- Liquidity Hub: a liquidity optimization and aggregation layer operating on top of DEX infrastructure.
- Perpetual Hub: an execution and infrastructure suite for intent-based perpetual futures.
- Orbs Agentic: an execution layer for autonomous DeFi agents.
- Orbs VM: a Docker-container-based decentralized execution service for more complex backend logic.
- Orbs Lambda: a simpler serverless-style decentralized execution service.
- Execution Services: the decentralized backend execution layer that includes Orbs VM and Lambda.
- Layer-3 (L3): the positioning term Orbs currently uses to describe itself as a supplementary execution layer for DeFi.
- CeFi-level execution: Orbs’ core message about bringing centralized-exchange-grade convenience and execution quality to DeFi.
- Multi-chain staking: an approach to staking that operates across multiple chain contexts such as Ethereum and Polygon.
- Hexa Foundation: an Orbs ecosystem or foundation context often referenced together with Netta Korin.
