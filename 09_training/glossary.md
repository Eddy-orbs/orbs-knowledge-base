# Glossary

- Orbs: DeFi용 advanced trading / execution 중심의 decentralized Layer-3 프로젝트로 현재 포지셔닝됨.
- ORBS token: Orbs 프로토콜의 핵심 토큰. fee settlement, staking, Guardian 선출, 보상에 연결됨.
- PoS: Proof-of-Stake. Orbs의 consensus / incentive 핵심 구조.
- Delegator: ORBS를 stake하고 Guardians에게 voting weight를 위임하는 참여자.
- Guardian: Orbs의 validator 역할을 수행하는 참여자 집합.
- dTWAP: decentralized time-weighted average price order. 큰 주문을 시간에 따라 쪼개 실행.
- dLIMIT: dTWAP의 특수한 경우로 설명되는 decentralized limit order 방식.
- dSLTP: decentralized stop-loss and take-profit order protocol.
- Liquidity Hub: DEX 위에서 작동하는 liquidity optimization / aggregation layer.
- Perpetual Hub: intent-based perpetual futures용 execution/infrastructure suite.
- Orbs Agentic: autonomous DeFi agents를 위한 execution layer.
- Orbs VM: Docker container 기반 decentralized execution service. 복잡한 backend logic 확장용.
- Orbs Lambda: 더 단순한 serverless 스타일 decentralized execution service.
- Execution Services: Orbs VM / Lambda를 포함한 decentralized backend execution 계층.
- Layer-3 (L3): Orbs가 현재 자신을 설명할 때 자주 사용하는 포지셔닝. DeFi 위의 supplementary execution layer 의미로 쓰임.
- CeFi-level execution: 중앙화 거래소 수준의 편의성과 실행 품질을 DeFi에 가져오려는 Orbs의 핵심 메시지.
- Multi-chain staking: Ethereum과 Polygon 등 여러 체인 맥락에서 staking 구조를 운용/확장하는 접근.
- Hexa Foundation: Netta Korin과 연결되어 언급되는 Orbs ecosystem/foundation 문맥 요소.
